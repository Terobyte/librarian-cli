# Librarian — спецификация v2.2 (инструкция к реализации)

Конвейер подготовки литературы для работы с LLM: любой входной формат → библиотека чистых Markdown-глав с каталогом. CLI — `lib`. Документ полностью заменяет v2.1: v2.2 — результат внешнего ревью v2.1 двумя независимыми агентами; вердикты и решения — в разделе 0-ter, все исправления вшиты в текст по месту.

**Ключевое требование не изменилось:** обработка полностью детерминированная. Без LLM, без сети, без случайности. Одинаковый вход всегда даёт побайтово идентичный выход (точное определение — раздел 2).

## 1. Задача и границы качества

Пользователь скармливает конвейеру книги и статьи. Конвейер раскладывает их на главы, вычищает форматный мусор, считает токены и ведёт каталог. Дальше LLM по каталогу выбирает и читает ровно те главы, которые нужны, — а не всю книгу.

Границы возможного без LLM (выводы v1 подтверждаются, переносятся без изменений):

- **EPUB, FB2, DOCX, HTML, TXT/MD — да, безусловно.** Структура записана в файле явно, мусор режется детерминированными правилами.
- **PDF с текстовым слоем — да, с оговоркой.** PDF хранит «буквы с координатами», структура восстанавливается эвристиками (раздел 7.2). На типографски обычных книгах результат чистый; сложная вёрстка — возможны дефекты.
- **Сканы и DJVU — нет.** Без OCR текст извлечь невозможно; конвейер честно сообщает «это скан, нужен OCR» и ничего не сохраняет.

Главное инженерное решение прежнее: стабильность обеспечивается не «идеальным парсером» (его не существует для PDF), а **контролем качества**. Каждая книга получает измеримый score (раздел 11), всё сомнительное помечается. Принцип — *fail loudly*.

**Не-цели** (без изменений): OCR, векторные базы и эмбеддинги, LLM-аннотации, сохранение картинок/формул/точной вёрстки таблиц, GUI.

**Периметр использования:** инструмент предназначен для личной библиотеки пользователя. DRM не обходится ни в каком виде (PDF под паролем без пустого user-пароля → `EncryptedError`, `failed`); ответственность за правомерность источников — на пользователе.

## 2. Контракт детерминизма

Определение. При фиксированных четырёх входах:

1. байты исходного файла,
2. эффективный конфиг (дефолты + config.toml + конфигурационные флаги CLI — их состав см. 13),
3. `PIPELINE_VERSION` (константа в коде),
4. lockfile зависимостей (точные версии pymupdf, trafilatura и пр.),

— содержимое каталога книги **побайтово идентично** между прогонами и машинами, **за единственным исключением** поля `provenance.ingested_at` в book.json.

Правила для кода, обеспечивающие контракт:

- `ingested_at` берётся из переменной окружения `SOURCE_DATE_EPOCH`, если она задана (тесты ставят `SOURCE_DATE_EPOCH=0` → `1970-01-01T00:00:00Z`, и тогда идентичность полная), иначе — из системных часов, UTC, ISO 8601 без миллисекунд.
- Запрещены в путях, влияющих на выход: `random`, `os.urandom`, `uuid4`, wall-clock (кроме provenance), **любой сетевой ввод-вывод** (в том числе скрытый в зависимостях — см. 6.0 про lxml и 10 про tiktoken), итерация по `set`/`dict` без `sorted(...)`, локале-зависимые операции (`str.lower` по локали, `locale.strcoll`). Регистронезависимость — только `str.casefold()`, сортировка — по кодпоинтам.
- **Хэш-рандомизация (проблема С-10):** `dict` в Python ≥ 3.7 упорядочен по вставке и от seed не зависит; собственный код не итерирует `set` без `sorted`. Для зависимостей гарантия эмпирическая: тест детерминизма (раздел 17) гоняется с двумя разными `PYTHONHASHSEED`; расхождение = баг зависимости → пин/обход + bump `PIPELINE_VERSION`.
- Канон сериализации — раздел 12.2. Все текстовые файлы: UTF-8, NFC, LF, один завершающий `\n`.
- Версии зависимостей пинуются lockfile-ом (`uv lock`); смена версии PyMuPDF может изменить извлечение → это трактуется как смена конвейера и требует bump `PIPELINE_VERSION`.
- `PIPELINE_VERSION` (строка, стартово `"2.2"`) повышается при **любом** изменении, влияющем на выходные байты. Golden-тесты (раздел 17) делают нарушение этого правила видимым.

## 3. Схема программы

### 3.1 Этапы

```
DETECT → EXTRACT → NORMALIZE → STRUCTURE → REFINE → VALIDATE → EMIT
```

1. **DETECT** — формат по магическим байтам (раздел 5).
2. **EXTRACT** — форматный экстрактор: файл → `RawDoc` (плоский поток блоков IR + метаданные + эталонный сырой текст для coverage). Под защитой лимитов 6.0.
3. **NORMALIZE** — блочные проходы: юникод/пробелы (все форматы) + PDF-вёрстка (раздел 7).
4. **STRUCTURE** — дерево секций из заголовков, выбор уровня нарезки, разрез на главы (раздел 8).
5. **REFINE** — секционные фильтры (раздел 9).
6. **VALIDATE** — метрики, score, статус `ok / review / failed` (раздел 11). *До* записи: `failed` не сохраняется.
7. **EMIT** — атомарная запись: главы, book.json, report.json, копия исходника; пересборка index.json (раздел 12).

### 3.2 Сквозной алгоритм `ingest_file` (и команды `ingest` в целом)

0. Пишущая команда на старте: берёт lock библиотеки (блокирующе, с таймаутом — 12.4), выполняет восстановление после падений (порядок в 12.4: сначала откат `.trash`, потом чистка).
1. `fmt = detect(path)`; неизвестный формат → `DetectError`, файл пропускается с сообщением.
2. `sha = sha256(файла)`; `cache_key = f"{sha}:{PIPELINE_VERSION}:{config_hash}"` (раздел 13).
3. Поиск `cache_key` — скан `provenance.cache_key` всех book.json библиотеки (на целевом масштабе ≤ ~5000 книг — миллисекунды; index.json ключей не хранит, он остаётся минимальным для LLM). Найден и нет `--force` → no-op, вернуть `skipped`.
4. `raw = extractor[fmt].extract(path, cfg)` — под лимитами 6.0; может подняться `ScanError`, `EncryptedError`, `BrokenFileError`, `LimitError`.
5. `ctx = DocContext(fmt, cfg, raw, report_draft)` — контекст несёт копилку вырезанного.
6. `blocks = apply_block_passes(raw.blocks, ctx)` — общие проходы N1–N3, затем PDF-проходы P1–P7 (порядок — раздел 7).
7. `root = build_tree(blocks)` → `L = choose_cut_level(root, cfg)` → `chapters = cut_chapters(root, L, cfg)`. Заголовков нет → fallback-раскрой (8.5) + структурный флаг.
8. `chapters = apply_section_passes(chapters, ctx)` — R1–R5, затем финальная нумерация (раздел 9). Токены внутри R-фильтров — **черновые** (по текстам блоков, см. 10).
9. **Финальный** подсчёт токенов каждой главы по её отрендеренному Markdown-тексту (раздел 10).
10. `metrics = compute_metrics(chapters, ctx)`; `score, status = score_and_status(metrics, cfg)`.
11. `status == failed` → report пишется в stderr с причиной, книга **не** сохраняется, вернуть `failed`.
12. **Разрешение идентичности (проблемы К-1, С-2):**
    - команда `reingest` → `book_id` известен заранее (имя существующего каталога), `make_id` не вызывается;
    - иначе ищется книга с тем же `source.sha256` в book.json библиотеки → найдена → её `book_id` переиспользуется (это переингест того же файла через `--force`);
    - иначе — новая книга: `book_id = make_id(meta, sha)` (раздел 12.1).
    Если для этого id существует старый book.json с `"meta_locked": true` → `title`/`author`/`lang` берутся из него, флаг переносится в новый book.json. Сборка `BookMeta` (включая `lang`-эвристику — 12.5).
13. `emit_book(...)` по протоколу публикации 12.4 (staging → trash → replace).
14. После обработки **всего пакета** (не после каждого файла — проблема С-7): `rebuild_index(lib_root)` — index.json пересобирается с нуля из всех book.json (обход `sorted` по имени каталога; dot-каталоги игнорируются; битые book.json пропускаются с предупреждением — 12.4).
15. Вернуть `IngestOutcome(id, status, score, warnings)`. CLI при нескольких файлах продолжает после ошибок и печатает итоговую таблицу (в stderr — это диагностика, раздел 15).

Любое неперехваченное исключение экстрактора конвертируется в `failed` с текстом причины — один битый файл не валит пакетную загрузку.

## 4. Модель данных (IR)

```python
class Format(str, Enum):
    FB2 = "fb2"; EPUB = "epub"; DOCX = "docx"; HTML = "html"
    TXT = "txt"; MD = "md"; PDF = "pdf"

class BlockKind(str, Enum):
    HEADING = "heading"; PARA = "para"; QUOTE = "quote"
    LIST_ITEM = "list_item"; TABLE = "table"; FOOTNOTE = "footnote"
    CODE = "code"; META = "meta"

@dataclass
class Block:
    kind: BlockKind
    text: str                       # NFC после NORMALIZE; \n внутри блока допустим
                                    #   (стихи, code, table-строки), пробелы схлопнуты
    level: int | None = None        # heading: 1..4 после нормализации уровней
    page: int | None = None         # только PDF, 1-based
    bbox: tuple[float, float, float, float] | None = None   # только PDF
    font_size: float | None = None  # только PDF, доминирующий по символам
    bold: bool = False              # только PDF
    origin: str = ""                # тег экстрактора для отладки: "p", "h2",
                                    #   "poem", "nav-title", "pattern:глава" …

@dataclass
class Section:                      # узел дерева после STRUCTURE
    title: str
    level: int                      # 0 — корень
    blocks: list[Block]             # контент до первого дочернего заголовка
    children: list["Section"]

@dataclass
class Chapter:
    n: int                          # присваивается в конце REFINE
    title: str                      # заголовок-путь (8.4), возможно "(2/5)"
    blocks: list[Block]
    tokens: int = 0                 # финальное значение — после рендера (шаг 9)

@dataclass
class RawDoc:
    fmt: Format
    blocks: list[Block]
    title: str | None               # из метаданных файла, иначе None
    author: str | None
    lang: str | None
    ref_text: str                   # эталон для coverage (11.1)
    pages: int | None = None        # PDF
    page_rects: list[tuple] | None = None

@dataclass
class DocContext:
    fmt: Format
    cfg: Config
    raw: RawDoc
    report: ReportDraft             # копилка: всё вырезанное, флаги, счётчики
```

Инварианты IR: после NORMALIZE весь `text` в NFC без zero-width/soft hyphen; `level` заполнен только у HEADING; `page/bbox/font_size` — только у блоков PDF; порядок списка `blocks` = порядок чтения.

## 5. Detect

По магическим байтам, не по расширению (единственное исключение — MD, у него магических байт нет). Проверки применяются строго сверху вниз, побеждает первая сработавшая:

| # | Признак | Формат |
|---|---|---|
| 1 | `%PDF` в первых 1024 байтах | PDF |
| 2 | ZIP: запись `mimetype` с содержимым `application/epub+zip` | EPUB |
| 3 | ZIP: запись `word/document.xml` (именно запись-файл — записей-каталогов в ZIP может не быть) | DOCX |
| 4 | ZIP: ровно одна запись `*.fb2` (прочие записи — обложки и пр. — игнорируются; ≥ 2 записей `*.fb2` → `DetectError`) | распаковать во временный буфер → FB2 |
| 5 | ZIP: иначе | `DetectError` |
| 6 | XML/текст: первый значимый тег — `<FictionBook` (см. правило сниффера ниже) | FB2 |
| 7 | XML/текст: первый значимый тег — `<!DOCTYPE html` или `<html` (без учёта регистра) | HTML |
| 8 | расширение `.md`/`.markdown` и файл текстовый (5.1) | MD |
| 9 | иначе, если декодируется как текст (5.1) | TXT |
| 10 | иначе | `DetectError` |

**Правило сниффера (шаги 6–7):** берутся первые 4 КБ; кодировка — по BOM, иначе как в 5.1; пропускаются BOM, пробельные символы, XML-декларация `<?xml …?>`, комментарии `<!-- … -->` и processing instructions; первый оставшийся тег и есть «первый значимый». Это покрывает XHTML и HTML с комментарием перед doctype.

Битый/зашифрованный ZIP → `BrokenFileError`. Распаковка — под лимитами 6.0.

5.1 «Текстовый»: charset-normalizer возвращает результат с уверенностью ≥ 0.5 и доля управляющих символов (кроме `\n\r\t`) < 1%.

## 6. Экстракторы

### 6.0 Харднинг и лимиты экстракции (проблемы К-2, С-5)

Входные файлы считаются **недоверенными**. Обязательные правила:

- **lxml** (FB2, EPUB, DOCX-выход mammoth, HTML-fallback): все парсеры создаются только через фабрику `xmlsafe.parser()` с `resolve_entities=False`, `no_network=True`, `load_dtd=False`, `huge_tree=False`. Это закрывает XXE (чтение локальных файлов через external entities), billion-laughs (экспоненциальное разворачивание сущностей) и скрытые сетевые обращения за DTD. Прямые вызовы `lxml.etree.parse`/`fromstring` без фабрики запрещены (проверяется grep-тестом в CI).
- **Размер исходника:** файл > `limits.max_source_mb` (256 МБ) → отказ с сообщением до извлечения.
- **ZIP (EPUB/DOCX/fb2.zip):** суммарный заявленный распакованный размер записей > `limits.zip_max_uncompressed_mb` (512 МБ) **или** коэффициент распаковки > `limits.zip_ratio_max` (100×) → `BrokenFileError("похоже на zip-bomb")`. Записи распаковываются потоково, с контролем фактического размера.
- **Таймаут извлечения:** `extract()` одного файла ограничен `limits.extract_timeout_s` (120 с); по истечении — `failed` с сообщением «извлечение зависло» для этого файла, пакет продолжается. Операционный guard: на выходные байты успешно обработанных книг не влияет.

Контракт экстрактора: (а) выдать блоки в порядке чтения, (б) заполнить `title/author/lang` из метаданных файла, если они там есть, (в) заполнить `ref_text` по правилу из 11.1, (г) не выполнять никакой чистки, кроме описанной в его подразделе (склейка строк — это восстановление абзацев, не чистка).

| Формат | Библиотека | Источник структуры |
|---|---|---|
| FB2 | lxml (через xmlsafe) | вложенность `<section>` / `<title>` |
| EPUB | ebooklib + lxml (через xmlsafe) | spine + `h1–h4`; fallback — файл-как-секция |
| DOCX | mammoth | стили Heading 1–4 → `h1–h4` |
| HTML | trafilatura | XML-вывод: `head[rend]`, `p`, `quote`, `item` |
| TXT / MD | своя логика | regex-паттерны глав (6.1.3) |
| PDF | PyMuPDF | эвристики раздела 7.2 |

### 6.1 TXT

1. **Кодировка**: `charset_normalizer.from_path(path).best()`. Обязательные тесты на cp1251 и koi8-r — русская литература в сети часто лежит именно в них. Не декодировалось → `BrokenFileError`.
2. **Абзацы**: разделитель — одна и более пустых строк. Внутри абзаца строки склеиваются:
   - строка кончается на `-` после буквы, следующая начинается со **строчной** буквы того же алфавита:
     - фрагмент после разрыва до первого небуквенного символа входит в `cfg.clean.keep_hyphen_suffixes` («то», «либо», «нибудь», «ка», «таки») → склейка **с сохранением** дефиса («кто-то»);
     - иначе → склейка без дефиса и без пробела (обычный перенос);
   - иначе — склейка через один пробел.
   Это единственное место (вместе с PDF), где живёт правило склейки переносов. **Принятый дефект:** перенесённое дефисное слово вне списка частиц («светло-/синий» → «светлосиний») без словаря не отличить от переноса; словарь означает недетерминизм между версиями — не делаем, документируем.
3. **Заголовки-кандидаты** — абзац из одной строки, совпадающий с одним из паттернов (все — в `cfg.chapters.patterns`, без учёта регистра):
   - `^(том|книга|book|volume)\s+([0-9ivxlc]+|[а-яёa-z]+)([.:—\-]\s*.{0,80})?$` → ранг 1
   - `^(часть|part|раздел)\s+…$` (та же хвостовая группа) → ранг 2
   - `^(глава|chapter)\s+…$` → ранг 3
   - `^([0-9]{1,4}|[IVXLC]{1,7})\.?$` (одиночное число/римская цифра) → ранг 3
   - строка без единой строчной буквы, ≥ 1 буква, длина ≤ 60 → ранг 3
   После сбора ранги сжимаются в непрерывные уровни 1..k с сохранением порядка (если в книге только «Глава N», она станет уровнем 1).
4. Остальные абзацы → `PARA`. Строки, начинающиеся с `>` (цитаты в письмах) не интерпретируются — TXT есть TXT.

### 6.2 MD

Собственный минимальный парсер (полный CommonMark не нужен). Построчные правила, в порядке приоритета:

1. **Front matter**: `---` в первой строке файла — всё до следующей строки `---` → один блок META.
2. **Fence**: ```` ``` ````-блоки → CODE (содержимое дословно, включая `\n`; инфо-строка после открывающего fence игнорируется).
3. **Setext**: строка только из `=` (или только из `-`), стоящая непосредственно под непустой строкой (без пустой строки между) → предыдущая строка становится HEADING уровня 1 (`=`) или 2 (`-`).
4. **ATX**: `#{1,4}` → HEADING уровня по числу решёток; `#{5,6}` → HEADING уровня 4.
5. **Thematic break**: строка только из ≥ 3 символов `-`, `*` или `_` (пробелы между ними допустимы), не съеденная правилом 3 → пропускается (не LIST_ITEM и не текст).
6. **Цитаты**: строки `> ` → QUOTE (подряд идущие объединяются в один блок, `\n` внутри).
7. **Списки**: `-`/`*`/`+`/`1.` в начале → LIST_ITEM (по одному блоку на пункт; вложенность сплющивается).
8. Прочее — PARA с правилами склейки строк из 6.1.2.

**Инлайн** (применяется к тексту всех блоков, кроме CODE): `[текст](url)` → `текст`; `![alt](url)` → `alt` (пустой alt — вставка исчезает); автоссылка `<http…>` → текст URL. Прочая инлайн-разметка (`*`, `**`, `` ` ``) не трогается — это авторский текст.

### 6.3 FB2

lxml (через xmlsafe), namespace-агностичный обход. Маппинг: `<title>` секции → HEADING, уровень = глубина вложенности `<section>`; `<p>` → PARA; `<epigraph>` → QUOTE (все параграфы эпиграфа объединяются, `\n` между ними); `<cite>` → QUOTE; `<poem>`: каждая `<stanza>` → PARA с `origin="poem"`, строки `<v>` соединены `\n`; `<subtitle>` → HEADING уровнем на 1 глубже текущей секции (макс 4); `<table>` → TABLE (ячейки строки через `\t`, строки через `\n`); `<binary>`, `<coverpage>` — пропускаются.

Метаданные: `title-info/book-title`, первый `<author>` — хранится как есть, в порядке `first-name last-name` («Имя Фамилия»), без сокращения до инициалов; `lang`.

**Сноски и комментарии:** основным считается первый `<body>` без атрибута `name`; **все остальные** `<body>` (`name="notes"`, `name="comments"` и любые прочие) в основной поток не попадают. Их секции собираются в отдельный список пар «номер — текст»; после основного контента добавляется синтетический HEADING уровня 1 с текстом `cfg.general.notes_chapter_title` («Примечания») и блоки сносок в формате `N. текст`. Инлайновые маркеры `<a type="note">` заменяются их текстовым содержимым в квадратных скобках: `[1]`; если содержимое уже начинается с `[` — скобки не дублируются.

### 6.4 EPUB

1. ebooklib: элементы spine в порядке spine; пропускаются item-ы со свойством `nav`, файлы, на которые ссылается `<guide>`/landmarks как cover/toc.
2. Каждый XHTML → lxml.html (через xmlsafe); обход body: `h1–h4` → HEADING уровня по тегу, `h5`/`h6` → HEADING уровня 4, `p` → PARA, `blockquote` → QUOTE, `li` → LIST_ITEM, `pre` → CODE, `table` → TABLE (ячейки `\t`, строки `\n`). Инлайновая разметка (`em`, `strong`, `a`) сплющивается в текст.
3. Границы файлов сами по себе **не** являются границами глав — заголовки сквозные по всему потоку.
4. **Fallback:** если после обхода всего spine найдено < 2 блоков HEADING, каждый spine-файл превращается в секцию: в начало его блоков вставляется HEADING уровня 1 с заголовком из nav/NCX. Сопоставление — по href **без фрагмента** (`chap01.xhtml#c1` → `chap01.xhtml`); несколько nav-записей на один файл → берётся первая. Нет записи в nav → заголовок = первые 60 символов первого PARA.
5. NCX/nav в основном режиме используется только как источник человеческих названий: если заголовок главы после нарезки пуст/односимволен, а nav для этой позиции даёт название — берётся из nav.

### 6.5 DOCX

`mammoth.convert_to_html()` → HTML → тот же маппинг, что в 6.4.2 (общий модуль `extractors/html_blocks.py`, парсинг через xmlsafe). mammoth выбран вместо python-docx, потому что превращает стили в семантический HTML — меньше ручного маппинга. Метаданные — из `docProps/core.xml` (title, creator). Fallback: ни одного HEADING → к PARA-блокам применяются паттерны 6.1.3 (общая функция).

### 6.6 HTML

`trafilatura.extract(html, output_format="xml", include_comments=False, include_tables=True, include_formatting=True)` — trafilatura сама отделяет основной контент от навигации и рекламы, для сохранённых статей это решает 90% чистки. Парсинг её XML: `<head rend="h2">` → HEADING (уровень из rend, нет rend → 2), `<p>` → PARA, `<quote>` → QUOTE, `<item>` → LIST_ITEM, `<code>` → CODE, `<row>/<cell>` → TABLE. **Неизвестные теги** → текст извлекается как PARA, тег считается в `report.unknown_tags` (не молча). Метаданные — `trafilatura.extract_metadata()`. Пустой результат extract → `BrokenFileError("не удалось выделить основной контент")`.

### 6.7 PDF

PyMuPDF. Порядок работы экстрактора (всё до проходов P1–P7):

1. `doc = fitz.open(path)`; `doc.needs_pass` → сначала `doc.authenticate("")` — зашифрованные пустым user-паролем PDF открываются штатно; не открылось → `EncryptedError`.
2. **Детект скана:** медиана длины `page.get_text("text")` по страницам < `cfg.pdf.scan_chars_per_page` (20) → `ScanError("текстовый слой отсутствует — это скан, нужен OCR")`.
3. По каждой странице `page.get_text("dict", sort=True)`. Каждый текстовый блок словаря → один `Block(kind=PARA)`:
   - строки внутри блока склеиваются по правилу переносов 6.1.2 (включая список частиц-суффиксов);
   - `font_size` — размер, доминирующий по числу символов, округлённый к ближайшим `cfg.pdf.size_round` (0.5) pt;
   - `bold = True`, если ≥ 60% символов имеют bold-флаг шрифта;
   - `page`, `bbox` заполняются.
4. `ref_text` = конкатенация `page.get_text("text")` всех страниц (эталон coverage).
5. `pages`, `page_rects` — в RawDoc. Метаданные — `doc.metadata` (title/author, пустые строки → None).

Никакой другой логики: заголовки, колонтитулы, сноски, колонки — это проходы P1–P7, чтобы каждый был тестируем отдельно.

## 7. NORMALIZE — блочные проходы

Проходы применяются строго в порядке таблицы; каждый — чистая функция `list[Block] → list[Block]`, всё удалённое складывается в `ctx.report`.

| Порядок | Проход | Форматы |
|---|---|---|
| 1 | N1 unicode | все |
| 2 | N2 whitespace | все |
| 3 | N3 controls | все |
| 4 | P1 page numbers | PDF |
| 5 | P2 headers/footers | PDF |
| 6 | P3 reading order | PDF |
| 7 | P4 defect pages | PDF |
| 8 | P5 headings | PDF |
| 9 | P6 cross-page merge | PDF |
| 10 | P7 footnotes | PDF |

### 7.1 Общие проходы

- **N1 unicode**: NFC; удалить U+00AD (soft hyphen), U+200B–U+200D, U+FEFF; `\r\n`/`\r` → `\n`.
- **N2 whitespace**: в каждой строке блока схлопнуть повторные пробелы/табы в один пробел (кроме блоков CODE и TABLE — там `\t` значим), срезать хвостовые пробелы; схлопнуть 3+ `\n` внутри блока до 2; пустые блоки удалить.
- **N3 controls**: управляющие символы C0/C1 (кроме `\n`, `\t`) → удалить, посчитав их в `report.control_chars` (пойдёт в encoding-метрику).

Правило v1-6.3 сохраняется дословно и распространяется на все проходы: **удалять можно только то, что порождено форматом, а не автором.** Никаких исправлений опечаток, типографики, кавычек, ударений.

### 7.2 PDF-проходы

- **P1 page numbers**: из текста блока срезаются пробелы и обрамляющая пунктуация (`— – - . ( ) [ ]`); остаток непуст, длиной ≤ `pagenum_max_chars` (4) и состоит только из цифр или римских цифр (любой регистр), центр bbox в верхней или нижней зоне `hf_zone` (10% высоты страницы) → удалить, счётчик в report.

- **P2 headers/footers**: сигнатура блока = текст, в котором каждая цифра заменена `#`, пробелы схлопнуты, `casefold()`. Для блоков в верхней/нижней зоне считается частота пар (зона, сигнатура). Пара встречается на ≥ `max(hf_min_pages, ceil(hf_page_ratio * pages))` страницах → все её блоки удаляются. Абсолютный минимум `hf_min_pages = 5` защищает короткие документы. Удалённые сигнатуры со списком страниц → report.

- **P3 reading order**: по каждой странице для «телесных» блоков (вне зон колонтитулов):
  1. x-центры блоков сортируются; ищутся разрывы между соседними центрами, удовлетворяющие колоночному критерию: разрыв ≥ `column_gap_ratio` (0.15) × ширины страницы **и** по обе стороны ≥ `column_min_share` (0.25) блоков страницы;
  2. ровно один такой разрыв → страница двухколоночная: сначала все блоки левой колонки сверху вниз (по y0), затем правой;
  3. **два и более** таких разрывов (три и более колонок — проблема С-11) → порядок остаётся как у `sort=True`, страница попадает в `report.multi_column_pages`; доля таких страниц > `multi_column_page_ratio` (10%) → жёсткий триггер review (11.2);
  4. иначе порядок остаётся как у `sort=True` (y0, затем x0).

- **P4 defect pages**: доля U+FFFD + непечатаемых на странице > `defect_char_ratio` (2%) → номер страницы в `report.pages_flagged` (влияет на encoding-метрику через сам текст; флаг — для `doctor`).

- **P5 headings** — детерминированная гистограмма вместо «кластеризации»:
  1. По всем PARA-блокам собирается гистограмма `font_size → число символов`. Размер тела `B` = размер с максимумом символов (при равенстве — меньший размер).
  2. Кандидаты-размеры: все размеры ≥ `heading_size_ratio` (1.15) × B, встречающиеся хотя бы в 2 блоках. Сортируются по убыванию, берутся первые `heading_max_levels` (3) → уровни 1..3.
  3. Блок становится HEADING уровня k, если его размер попал в уровень k, текст занимает ≤ 2 строк и ≤ `heading_max_chars` (120) символов, и блок не оканчивается на `.` или `,`.
  4. Bold-правило: блок размера B, `bold=True`, одна строка ≤ `bold_heading_max_chars` (60) символов, без точки в конце → HEADING уровня (число размерных уровней + 1), максимум 4.
  5. Если размерных уровней не нашлось вовсе (моношрифтовый PDF): к блокам применяются паттерны 6.1.3 — общий fallback.
  6. Два подряд идущих HEADING одного уровня на одной странице с зазором по y < 1.5 высоты строки склеиваются в один (многострочные заголовки). **Высота строки** блока = высота его bbox / max(1, число строк в блоке).

- **P6 cross-page merge**: последний PARA страницы не оканчивается на `. ! ? … : ; » " )`, а первый PARA следующей страницы начинается со строчной буквы или тире продолжения → блоки склеиваются (с обработкой переноса по 6.1.2). Применяется после P2/P3, когда колонтитулы уже не мешают.

- **P7 footnotes**: блок с размером < `footnote_size_ratio` (0.85) × B, верх bbox в нижней `footnote_zone` (трети) страницы, текст начинается с цифры или `*` → `kind=FOOTNOTE`. Сноски не удаляются из потока и **сохраняют свою позицию среди блоков вплоть до EMIT** (см. 9 о поведении при R3/R4); лишь при рендере главы они выводятся в её конце под разделителем `---`, в порядке следования. Маркеры-надстрочники в теле текста не трогаются и не линкуются (осознанное упрощение). `cfg.pdf.footnotes = "keep" | "drop"`.

## 8. STRUCTURE

### 8.1 Нормализация уровней

Множество реально встретившихся уровней HEADING переиндексируется в плотное 1..k с сохранением порядка (были только 2 и 4 → станут 1 и 2). Документ, начинающийся с h2, не должен получать фантомный пустой уровень 1.

### 8.2 Построение дерева

Классический стек: идём по блокам; HEADING уровня L закрывает все секции уровня ≥ L и открывает новую; не-HEADING блоки складываются в текущую секцию. Блоки до первого заголовка → секция уровня 1 с заголовком `cfg.general.preface_title` («Начало»).

### 8.3 Выбор уровня нарезки (auto-deepen)

```
L = cfg.chapters.cut_level_start          # 2
повторять:
    сегменты = разрез потока по заголовкам уровня ≤ L
    если median(токены сегментов) > cfg.chapters.deepen_median   # 12000
       и существуют заголовки уровня L+1
       и L < 4:
           L += 1; продолжить
    иначе: стоп
если L == cut_level_start и median < cfg.chapters.shallow_median  # 500
   и L > 1:
       L = 1  (пересчитать сегменты)
```

Токены сегмента здесь — **черновые**: tiktoken по текстам блоков сегмента, соединённым `\n\n` (раздел 10). «Война и мир»: L=2 даёт «Части» с медианой ~60k > 12000, есть уровень 3 → L=3, медиана ~3k → стоп: главами становятся настоящие главы. Сборник рассказов с редкими подзаголовками: медиана нормальная → остаётся L=2. Документ из сплошных мелких подзаголовков: медиана < 500 → откат на L=1.

### 8.4 Заголовок-путь

Заголовок главы = названия заголовков-предков от уровня 1 до уровня разреза включительно, соединённые « · », подряд идущие дубли схлопнуты. Для «Войны и мира»: `Том первый · Часть первая · I` — именно это видит LLM в каталоге, а не голое «I». Первая строка файла главы — этот же заголовок-путь.

### 8.5 Fallback без заголовков

Заголовков нет (вся книга — один сегмент) → **структурный красный флаг**: статус не выше `review` (обеспечивается structure-категорией 0.3 — при весах из 14 score не может превысить 0.825), текст режется на части по `cfg.chapters.fallback_part_tokens` (~6000) токенов **строго по границам блоков** (блок неделим), заголовки частей — `«{title} (i/k)»`.

## 9. REFINE — секционные фильтры

Порядок фиксирован: R1 → R2 → R3 → R4 → R5 → нумерация. Токены во всех фильтрах — **черновые** (по текстам блоков, раздел 10): финальный подсчёт по отрендеренному Markdown происходит после REFINE (шаг 9 конвейера).

**FOOTNOTE при перестройках (проблема С-3):** сноски — обычные блоки потока до самого EMIT. R3 переносит их вместе с остальными блоками поглощаемой главы; R4 оставляет каждую сноску в той части, куда она попала по позиции в потоке — маркер и сноска с одной страницы физически соседствуют и оказываются в одной части. Никакой отдельной маршрутизации сносок нет — и не нужно.

- **R1 служебные секции**: глава короче `meta_max_tokens` (150) токенов, содержащая любой маркер из `cfg.clean.meta_markers` («ISBN», «©», «Все права защищены», «All rights reserved», «Литагент») → целиком в report (`removed.meta_sections`, с полным текстом), из библиотеки исключается. Ничего не исчезает бесследно.

- **R2 оглавление**: глава размером ≤ `toc_max_tokens` (2000) токенов удаляется как печатное оглавление, если (а) > `toc_numeric_line_ratio` (60%) её строк оканчиваются числом, **или** (б) ≥ `toc_heading_dup_ratio` (80%) её строк текстуально совпадают (casefold, схлопнутые пробелы) с **исходными заголовками секций** других глав (не с заголовками-путями — печатное оглавление содержит именно исходные названия). Структуру мы строим сами, печатное оглавление — мусор. Текст → report.

- **R3 склейка мелочи**: один проход слева направо по списку глав. Глава < `tiny_tokens` (30) токенов (эпиграф, посвящение) → её блоки приклеиваются в начало следующей главы; если она последняя — в конец предыдущей. Заголовок поглощаемой главы понижается до HEADING внутри принявшей (уровень: разрезной + 1). Принявшая глава дальше участвует в проходе уже с новым размером.

- **R4 резка гигантов**: глава > `max_tokens` (8000):
  1. внутри есть заголовки глубже уровня разреза → уровни перебираются от «разрезной + 1» вглубь; берётся первый (самый неглубокий), при котором все куски ≤ `max_tokens`; такого нет → берётся самый глубокий, а куски, оставшиеся > `max_tokens`, дорезаются механически по п. 2; заголовки-пути обновляются;
  2. иначе — механическая резка: блоки набираются до `part_target_tokens` (6000), рез только между блоками; заголовки частей `«{title} (i/k)»`, файлы `NNN-slug-p1.md`, `-p2.md`… (NNN у каждой части свой; суффикс `-pK` сохраняется для читаемости).
  3. **Исключение из «блок неделим»:** одиночный блок > `max_tokens` (стенограмма, юридический абзац) режется принудительно: PARA/QUOTE — по границам предложений (`. ! ?` + пробел), CODE/TABLE — по строкам, на куски ≤ `part_target_tokens`; каждое событие — в `report.oversize_blocks_split`. Иначе `get --budget` на таких документах не работал бы никогда.
  Каждая часть — **отдельная запись каталога** со своим `n` и своими токенами: LLM должна уметь запросить именно вторую половину главы.

- **R5 пустые**: главы без блоков (всё съели фильтры) удаляются.

- **Нумерация**: сквозная 1..N по финальному порядку, в имени файла — три цифры с ведущими нулями.

## 10. Токены

tiktoken, энкодинг `o200k_base`, имя зафиксировано в конфиге.

**Офлайн, полная сборка энкодера (проблемы К-3, К-4):** штатный `tiktoken.get_encoding()` при первом вызове скачивает BPE-файл из сети — это нарушало бы контракт «без сети». Поэтому:

- файл `o200k_base.tiktoken` (mergeable_ranks) вендорится в пакет (`src/librarian/assets/`), включается в sdist/wheel через package-data и читается через `importlib.resources` (не по путям файловой системы — иначе сломается zip-import);
- его **sha256 зашит константой** в tokens.py и проверяется при загрузке; несовпадение → `LibError("повреждён словарь токенизатора")` — тихо врущий счётчик хуже падения;
- `pat_str` и `special_tokens` для `o200k_base` **зафиксированы константами** в tokens.py (скопированы из `tiktoken_ext/openai_public.py` соответствующей версии) — BPE-файла одного недостаточно для сборки энкодера;
- энкодер собирается напрямую: `tiktoken.Encoding(name=..., pat_str=..., mergeable_ranks=..., special_tokens=...)`. Рантайм не делает ни одного сетевого запроса.

Два режима подсчёта:

- **Черновой** (8.3, R1–R4): по текстам блоков главы/сегмента, соединённым `\n\n`. Дёшево, достаточно точно для порогов.
- **Финальный** (шаг 9 конвейера): по финальному Markdown-тексту главы (включая строку заголовка и перенесённые сноски) — ровно то, что получит LLM через `lib get`. В book.json/каталог идут только финальные значения; `total_tokens` книги = сумма по главам.

Расхождение с токенизаторами других моделей ±10% — для бюджетирования достаточно.

## 11. VALIDATE

### 11.1 Эталон для coverage

| Формат | Эталонное извлечение |
|---|---|
| PDF | конкатенация `page.get_text("text")` (PyMuPDF, без poppler) |
| EPUB | text_content всех spine-документов |
| FB2 | `itertext()` всех `<body>` |
| DOCX | `mammoth.extract_raw_text()` |
| HTML | plain-text вывод trafilatura (сырой HTML эталоном быть не может — coverage наказывал бы за вырезанную навигацию) |
| TXT/MD | декодированный файл |

`coverage = tokens(все главы) / max(1, tokens(эталон) − draft_tokens(removed.meta_sections) − draft_tokens(removed.toc))` — токены секций, **легитимно** удалённых R1/R2 (их полный текст лежит в report), вычитаются из знаменателя: метрика не должна наказывать за собственные задокументированные правила.

### 11.2 Метрики

| Метрика | Формула | Жёсткий триггер review |
|---|---|---|
| coverage | п. 11.1 | < 0.60 или > 1.02 |
| garbage_ratio | доля «мусорных» строк среди непустых строк финального Markdown всех глав: strip-длина ≤ 2 (кроме `— – * ―`), либо одинокое число ≤ 4 цифр отдельным абзацем вне LIST_ITEM | > 2% |
| encoding_score | (U+FFFD + вхождения маркеров из `cfg.quality.mojibake_markers`: `Ð, Ñ, Ã, â€, пїЅ` + удалённые в N3 управляющие) / всего символов | > 0.5% |
| structure | категория: 1.0 — главы найдены, медиана ∈ [300, 20000] токенов; 0.7 — найдены, медиана вне диапазона; 0.3 — глав нет, применён fallback 8.5 | главы не найдены |
| dehyphen_residue | доля строк выхода, оканчивающихся дефисом после буквы; **считается только для TXT/MD/PDF** — форматов со склейкой переносов; для остальных метрика = 0 (субоценка 1), иначе стихи в FB2/EPUB давали бы ложный review | > 1% |
| layout (только PDF, без веса в score) | — | `multi_column_pages` > 10% страниц |

### 11.3 Субоценки [0, 1] — кусочно-линейно

| Метрика | s = 1 при | s = 0 при | между — линейно |
|---|---|---|---|
| coverage | 0.70 ≤ c ≤ 1.02 | c ≤ 0.40 или c ≥ 1.15 | да, с обеих сторон |
| garbage | g ≤ 0.5% | g ≥ 5% | да |
| encoding | e = 0 | e ≥ 2% | да |
| structure | значение категории напрямую | — | — |
| dehyphen | d ≤ 0.2% | d ≥ 3% | да |

### 11.4 Score

`score = 0.30·s_cov + 0.25·s_struct + 0.20·s_garb + 0.15·s_enc + 0.10·s_dehyph` (веса — в конфиге).

### 11.5 Статус

- `failed`: score < 0.60, либо фатальная ошибка (`ScanError`, `EncryptedError`, `BrokenFileError`, `LimitError`). Не сохраняется; причина — человеческим языком.
- `review`: score < 0.90 **или** сработал любой жёсткий триггер из 11.2. Сохраняется + жёлтое предупреждение.
- `ok`: иначе. Молча в библиотеку.

## 12. EMIT

### 12.1 Идентификаторы и слаги

Транслитерация — фиксированная таблица в коде (не внешняя библиотека — версии меняют выход):

```
а a  б b  в v  г g  д d  е e  ё e  ж zh  з z  и i  й y  к k  л l  м m  н n
о o  п p  р r  с s  т t  у u  ф f  х h  ц ts  ч ch  ш sh  щ sch  ъ ∅  ы y
ь ∅  э e  ю yu  я ya
```

`slugify(s, max_len)`: NFC → `casefold` → транслит → всё, кроме `[a-z0-9]`, в `-` → схлопнуть повторные `-` → обрезать по краям → усечь до `max_len` по границе `-` → пусто → `"text"`.

`book_id = slugify(f"{author} {title}", max_len=cfg.slug.max_len)`, нет метаданных → слаг имени файла без расширения. Каталог с таким id уже существует и его `source.sha256` отличается → суффикс `-` + первые 6 hex sha256 нового файла. Имя файла главы: `{n:03d}-{slugify(title, max_len=cfg.slug.chapter_len)}.md` (+ `-pK` для частей) — усечение всегда внутри `slugify`, по границе `-`; голых срезов `[:50]`, оставляющих висячий дефис, нет.

### 12.2 Канон сериализации

JSON: `json.dumps(obj, ensure_ascii=False, indent=2, sort_keys=True)` + `"\n"`. Markdown/тексты: UTF-8, NFC, LF, без хвостовых пробелов, один завершающий `\n`. `ingested_at` — единственное недетерминированное поле, изолировано в `provenance` (раздел 2).

### 12.3 IR → Markdown

| Kind | Вывод |
|---|---|
| HEADING (разрезной) | `# {заголовок-путь}` — первая строка файла |
| HEADING (внутренний, уровень L+k) | `{"#" * min(k+1, 6)} {текст}` |
| PARA | текст (внутренние `\n` — стихи — сохраняются) + пустая строка |
| QUOTE | каждая строка с префиксом `> ` |
| LIST_ITEM | `- {текст}` |
| CODE | fence из бэктиков; длина fence = max(3, самый длинный ран бэктиков в содержимом + 1) |
| TABLE | pipe-таблица из `\t`/`\n`-разметки: первая строка — заголовок, после неё вставляется разделитель `\|---\|…`; каждый символ `\|` в тексте ячейки выводится как **два символа: обратная косая черта + вертикальная черта**; выравнивание не восстанавливается |
| FOOTNOTE | все блоки главы — в конце, после строки `---`, в исходном порядке |
| META | не выводится никогда (живёт в report) |

Текст остальных блоков не экранируется: потребитель — LLM, а не Markdown-рендерер; это осознанное решение.

Файл главы — чистый Markdown **без frontmatter**: он скармливается LLM как есть, вся мета живёт в book.json.

### 12.4 Раскладка, атомарность, блокировка

```
library/
  index.json
  tolstoy-voyna-i-mir/
    book.json
    report.json
    source/voyna.pdf          # по умолчанию
    chapters/
      001-tom-pervyy-chast-pervaya-i.md
      002-….md
```

**Протокол публикации** (`os.replace` не умеет заменять непустой каталог — ENOTEMPTY на POSIX, ошибка на Windows, поэтому «просто заменить» нельзя):

1. Книга собирается целиком в `library/.staging/<id>/`.
2. Целевой каталог `library/<id>/` существует (только при `--force` или реингесте) → `os.replace(library/<id>, library/.trash/<id>)`.
3. `os.replace(library/.staging/<id>, library/<id>)` — атомарный rename в пределах одной ФС.
4. `library/.trash/` удаляется (`shutil.rmtree`).

**Восстановление после падений (проблема С-8)** — выполняется каждой пишущей командой сразу после взятия lock, строго в этом порядке:

1. для каждого `library/.trash/<id>`: если `library/<id>` **не существует** → `os.replace(.trash/<id>, library/<id>)` — откат публикации, упавшей между шагами 2 и 3; старая версия книги возвращается на место, данные не теряются;
2. остатки `library/.staging/` удаляются (недособранные книги);
3. остатки `library/.trash/` удаляются (замещённые старые версии, чья публикация завершилась).

Порядок принципиален: чистка `.trash` раньше отката уничтожала бы единственную копию книги.

**Блокировка (проблема С-9):** любая пишущая команда (`ingest`, `rm`, `reingest`) берёт эксклюзивный lock на `library/.lock`: `fcntl.flock` (POSIX) / `msvcrt.locking` (Windows). Ожидание блокирующее, с таймаутом `general.lock_timeout_s` (30 с) → «библиотека занята другим процессом», exit 1. Известное ограничение: на сетевых ФС (NFS/SMB) advisory-lock ненадёжен — библиотека на сетевом диске официально не поддерживается.

**index.json (проблемы С-1, С-4, С-7):** пересобирается целиком из всех book.json (обход `sorted` по имени каталога; dot-каталоги игнорируются) — **один раз на команду**, после обработки всего пакета. Запись атомарная: tmp-файл в том же каталоге + `os.replace` — читатели (`list`/`get`/`info`) lock не берут, им гарантирована целостность атомарной заменой. Битый book.json → warning в stderr, книга пропускается в индексе (и подсвечивается в `doctor`) — одна ручная ошибка не кладёт всю библиотеку. Целевой масштаб — до ~5000 книг: скан и пересборка укладываются в сотни миллисекунд, индекс-кэши не нужны.

  "title": "Война и мир",
  "author": "Лев Толстой",
  "lang": "ru",
  "meta_locked": false,
  "source": {"file": "voyna.pdf", "format": "pdf", "sha256": "…"},
  "provenance": {
    "ingested_at": "2026-07-05T12:00:00Z",
    "pipeline_version": "2.2",
    "config_hash": "9f3a…",
    "cache_key": "…sha256…:2.2:9f3a…"
  },
  "quality": {"status": "ok", "score": 0.94},
  "total_tokens": 812345,
  "chapters": [
    {
      "n": 1,
      "file": "chapters/001-tom-pervyy-chast-pervaya-i.md",

```

`provenance.cache_key` — ключ из раздела 13; по нему шаг 3 конвейера находит уже загруженные книги. `source.sha256` — по нему шаг 12 находит существующую книгу при повторном ingest.

`summary` — **без LLM**: первый PARA главы длиной ≥ 15 токенов, обрезка до 300 символов по границе слова с «…»; такого PARA нет → первый непустой блок любого вида; глава пуста → пустая строка. Если внутри главы есть подзаголовки — добавляются через ` — ` первые 8 названий, соединённые « · ». Для частей `(i/k)` при i ≥ 2 — первый абзац этой части.

`lang`: из метаданных файла; их нет → детерминированная эвристика по буквам всего текста: ≥ 50% букв — кириллица → `"ru"`; ≥ 50% — латиница → `"en"`; иначе `null`.

`author`/`title`/`lang` можно править руками: пользователь ставит `"meta_locked": true`, и любой последующий реингест переносит эти поля и сам флаг из старого book.json (шаг 12 алгоритма — проблема С-2).

## 13. Кэш и идемпотентность

`cache_key = f"{sha256(исходника)}:{PIPELINE_VERSION}:{config_hash}"`. `config_hash` = sha256 канонического JSON эффективного конфига: дефолты + config.toml + **конфигурационные флаги CLI** — на сегодня это ровно два: `--config` (содержимое файла) и `--no-keep-source` (маппится в `Config.keep_source`; он входит в хэш, потому что меняет состав каталога книги). Разовые флаги запуска (`--force`, `--library`, `--verbose`) выходных байт книги не меняют и в хэш не входят. Ключ хранится в `provenance.cache_key`; повторный `ingest` того же файла при совпадении ключа — no-op без `--force`.

Исходник копируется в `source/` **по умолчанию**; `--no-keep-source` отключает (и делает книгу непригодной для реингеста). `lib reingest --all` пересборка библиотеки из сохранённых исходников текущим кодом/конфигом; книги без исходника пропускаются с предупреждением.

**Стабильность идентичности (проблема К-1):** id книги назначается один раз. Реингест получает id явно (имя каталога); повторный `ingest --force` того же файла находит книгу по `source.sha256` и переиспользует её id; `make_id` вызывается только для действительно новых книг (шаг 12 алгоритма). Ссылки пользователя на `book-id` не бьются никогда; изменившиеся `title`/`author` обновляются внутри book.json (с учётом `meta_locked`).

## 15. CLI

Первое действие CLI (проблема С-6): `sys.stdout.reconfigure(encoding="utf-8", newline="\n")` и то же для stderr — на Windows дефолтная консольная кодировка (cp1251 и пр.) ломала бы контракт «побайтово идентично между машинами».

Данные — в **stdout**, диагностика, таблицы прогресса и предупреждения — в **stderr** (вывод `get` пайпится в чат с LLM, его нельзя загрязнять). Таблицы `list`/`info` — это данные соответствующих команд → stdout (через rich). Сводная таблица после `ingest` — диагностика → stderr.

| Команда | Поведение |
|---|---|
| `lib ingest <path…> [--force] [--no-keep-source] [--config cfg.toml]` | пакетная загрузка; после — сводная таблица id/status/score (stderr); ошибки не прерывают пакет |
| `lib list` | таблица книг из index.json: id, автор, название, глав, токенов, статус |
| `lib list <book-id>` | оглавление: n, заголовок-путь, токены, summary (обрезан до ширины) |
| `lib get <book-id> <spec>` | главы в stdout, соединённые `\n\n` |
| `lib get <book-id> --budget 12000 [--from K]` | жадно подряд главы начиная с K (деф. 1), пока сумма токенов ≤ бюджета; что не влезло — в stderr; не влезла даже первая → stderr-сообщение, exit 1 |
| `lib info <book-id>` | book.json + метрики/субоценки из report.json |
| `lib doctor [book-id]` | без id — все книги в review + каталоги с битым book.json; с id — что вырезано (removed.*), флаги страниц, сработавшие триггеры |
| `lib rm <book-id>` | удалить каталог книги, пересобрать индекс |
| `lib reingest --all [--config …]` | пересборка из `source/` текущим кодом |

**Грамматика `<spec>`:** `spec = range("," range)*`; `range = N | N-M`, номера 1-based, `M ≥ N`. Примеры: `3`, `2,5-7`, `1-3,9`. Номер вне диапазона 1..N книги или `M < N` → сообщение в stderr, exit 1. **`<spec>` и `--budget` взаимоисключающие:** заданы оба → ошибка использования, exit 2.

Коды выхода: `0` — успех (включая review: это успех с предупреждением); `1` — хотя бы одна ошибка выполнения (failed-книга, неизвестный id, номер главы вне диапазона, невыполнимый бюджет, занятая библиотека); `2` — ошибка использования (typer, spec×budget). Глобальный флаг `--library <dir>`; приоритет: флаг → `$LIB_HOME` → `./library`.

## 16. Ошибки

```python
class LibError(Exception): ...
class DetectError(LibError): ...          # неизвестный формат
class ExtractError(LibError): ...
class ScanError(ExtractError): ...        # PDF без текстового слоя
class EncryptedError(ExtractError): ...   # PDF под паролем (пустой пароль уже испробован)
class BrokenFileError(ExtractError): ...  # битый zip/xml, недекодируемый текст, zip-bomb
class LimitError(ExtractError): ...       # превышен лимит 6.0 (размер, таймаут)
class UnknownBookError(LibError): ...     # id не найден (get/info/rm)
```

`ScanError`/`EncryptedError`/`BrokenFileError`/`LimitError` → статус `failed` с их сообщением; `DetectError` → пропуск файла; всё остальное неожиданное — тоже `failed` c текстом исключения (пакет не падает), плюс полный traceback в stderr при `--verbose`.

## 17. Тестирование

- **Фикстуры**: 2–3 реальных файла на формат; обязательно: TXT в cp1251 и koi8-r, TXT с переносами и дефисными словами («кто-то» на границе строки), MD с front matter, setext-заголовками и thematic break, FB2 со сносками и стихами, **FB2 с external entity в DTD (тест XXE: сущность не разворачивается)**, fb2.zip с обложкой внутри, EPUB без внутренних заголовков (глава = файл), EPUB с nav-ссылками вида `file.xhtml#anchor`, DOCX без стилей заголовков, двухколоночный PDF, **трёхколоночный PDF (уходит в review)**, PDF с «Том/Часть/Глава», PDF с пустым user-паролем, зашифрованный PDF, PDF-скан (5 страниц достаточно), **синтетический zip-bomb (даёт failed, не OOM)**.
- **Golden-тесты**: для каждой фикстуры эталонная библиотека зафиксирована в `tests/golden/<name>/`; тест гоняет ingest с `SOURCE_DATE_EPOCH=0` и сравнивает деревья побайтово. Регенерация — `scripts/update_golden.py`, любой diff — осознанное решение в code review, а не сюрприз. CI гоняет тесты **с отключённой сетью** — скрытые сетевые обращения (tiktoken, DTD lxml) всплывают сразу.
- **Тест детерминизма**: два прогона одной фикстуры в два каталога (`SOURCE_DATE_EPOCH=0`), **с разными `PYTHONHASHSEED` (0 и 42)** → рекурсивное побайтовое сравнение (проблема С-10: ловит set-итерации в зависимостях).
- **Тест идемпотентности кэша**: второй `ingest` того же файла → `skipped`, библиотека не изменилась побайтово.
- **Тест восстановления**: искусственное прерывание между шагами протокола 12.4 (мок os.replace) → следующий запуск возвращает библиотеку в целостное состояние, книга не теряется (проблема С-8).
- **Тест установки**: сборка wheel → установка в чистый venv → `lib ingest` работает (ассет токенизатора попал в пакет — проблема К-4). Grep-тест: в коде нет прямых `lxml.etree.parse`/`fromstring` мимо `xmlsafe` (проблема К-2).
- **Юнит на каждое правило** N1–N3, P1–P7, R1–R5 — на минимальном синтетическом IR (для PDF-проходов блоки конструируются руками с bbox/размерами, без реального PDF).
- **Свойства**: идемпотентность нормализации (проход дважды == однажды); резка R4 сохраняет мультимножество блоков (кроме документированного исключения — принудительной резки oversize-блока, где сохраняется конкатенация текста); нумерация глав плотная 1..N.
- **CI-матрица**: ubuntu-latest, macos-latest, windows-latest × Python 3.11 и 3.13; golden сравниваются между ОС (это и есть проверка «между машинами»).
- **Перф-смоук**: PDF на 500 страниц укладывается в 30 с на CI-раннере (мягкий порог, warning).

## 19. Стек

Python ≥ 3.11. Зависимости: `pymupdf`, `ebooklib`, `lxml`, `mammoth`, `trafilatura`, `charset-normalizer`, `tiktoken`, `typer`, `rich`. Никакого ML, никакой сети — в том числе скрытой: BPE-словарь tiktoken вендорится в пакет (раздел 10), lxml не ходит за DTD (раздел 6.0). Установка `pipx install`. Ориентир: PDF на 500 страниц — до 30 секунд на обычном ноутбуке. Поддерживаемые ОС: Linux, macOS, Windows (CI-матрица — раздел 17); библиотека на сетевых ФС не поддерживается (12.4).

## 20. v3 (за рамками этой спеки)

Без изменений: OCR сканов (ocrmypdf/tesseract) с пометкой «распознано, возможны ошибки»; MOBI/AZW3 через calibre CLI; `--summarize` — LLM-аннотации опциональным флагом; `lib serve` — MCP-сервер, чтобы LLM сам дёргал каталог и главы без копипасты.
