# Librarian `lib verify` — проверка и атрибуция цитат — Design

**Дата:** 2026-07-08. **База:** 0.1.1 (M7 отгружен: PyPI, MCP registry), спека v2.2,
M1–M7 закрыты.

## 1. Мотивация

LLM, цитирующий книгу, регулярно перевирает: склеивает фразы, «улучшает» формулировки,
приписывает не тому автору. У librarian уже есть всё, чтобы это ловить детерминированно:
побайтово стабильные главы + FTS5-индекс. `lib verify` закрывает нишу, которой нет ни у
одного book-MCP из конкурентного обзора M6 (2026-07-07: ebook-mcp, calibre-mcp,
book-mcp-server): **побайтовая проверка цитаты по первоисточнику**, без LLM и сети.
Перед анонсом перепроверить свежим поиском.

Два режима одной команды:

- **Проверка** (`--book` указан): «эта цитата действительно есть в этой книге?» —
  полный скан книги, recall 100%.
- **Атрибуция** (без `--book`): «откуда эта цитата?» — поиск по всей библиотеке через
  FTS5-кандидатов.

Позиционирование для README/анонса: *"An MCP server that catches Claude misquoting
your books."*

## 2. Принцип: ноль изменений в конвейере

`verify` — читатель готового выхода, как `find`/`get`. Конвейер ingest не знает о его
существовании.

- `PIPELINE_VERSION` не меняется; golden не регенерируются; проверка каждой задачи —
  `uv run pytest -q` зелёный без изменения golden.
- Версия пакета → 0.2.0 (новая фича, обратная совместимость полная).

```
library/ (готовые главы) ──→ verify.py ──→ вердикт + локация + diff
        ↑                        ↑
  режим книги: полный скан   режим полки: search.py (FTS5) → топ-K глав-кандидатов
```

## 3. Ядро: `src/librarian/verify.py`

Чистые функции, без состояния, по образцу `catalog.py`. Публичный вход один:

```python
def verify_quote(lib_root: Path, quote: str, *,
                 book_id: str | None = None, limit: int = 3) -> dict
```

### 3.1 Нормализация (цитата и текст глав — одинаково)

NFC → вычистка markdown-обвязки (строки-заголовки `#…`, маркеры списков) → casefold →
ё→е → типографика в ASCII (кавычки «»""„, тире —–, многоточие …) → схлопывание
пробельных прогонов в один пробел. Ведётся **карта смещений** нормализованный→оригинал,
чтобы в ответе показывать подлинный текст книги, а не нормализованный.

### 3.2 Поиск совпадения

**Режим книги:** для каждой главы — скользящее окно по словам шириной
`len(quote_words)` с шагом `max(1, len//4)`; `difflib.SequenceMatcher.quick_ratio()`
как дешёвый префильтр, точный `ratio()` — только для окон выше порога префильтра;
у лучшего окна границы уточняются (расширение/сужение по словам, пока `ratio` растёт).
`difflib` — stdlib, детерминирован.

**Режим полки:** значимые слова цитаты (стоп-слова долой, стемминг — переиспользуем
`_stem_word` из `search.py`) → FTS5 MATCH (OR-семантика, ранжирование bm25) → топ-20
глав-кандидатов → тот же оконный проход только по ним. Перед запросом — ленивый
`sync()` (паттерн `find`). Цитата **короче 5 значимых слов** — отказ (usage error):
«слишком короткая цитата для поиска по библиотеке — укажи книгу через --book».

**Детерминизм ничьих:** равный `similarity` → сортировка (book_id, номер главы,
смещение окна). Два одинаковых вызова обязаны давать побайтово идентичный JSON.

### 3.3 Вердикты

| verdict | порог | смысл |
|---------|-------|-------|
| `exact` | ratio = 1.0 | совпадение после нормализации (типографика не в счёт) |
| `close` | ≥ 0.95 | почти точная, расхождения показаны word-diff'ом |
| `distorted` | ≥ 0.75 | это место, но цитата переврана |
| `not_found` | < 0.75 | в книге/библиотеке такого нет |

Пороги 0.95/0.75 — стартовые, калибруются на фикстурах при реализации (меняются
константами в `verify.py`, в контракт не зашиты). Вердикт ответа = вердикт лучшего
совпадения.

### 3.4 Word-diff расхождений

Для `close`/`distorted`: `SequenceMatcher.get_opcodes()` по спискам слов
(нормализованным), каждая не-`equal` группа → пара `{quoted: "...", source: "..."}`
(пустая строка при вставке/удалении). Показ — по оригинальным словам через карту
смещений.

## 4. Контракты

### 4.1 Структура результата (единая для CLI `--json` и MCP)

Canonical-JSON (как весь проект):

```json
{
  "verdict": "close",
  "matches": [
    {
      "book_id": "bulgakov-master-i-margarita",
      "book_title": "Мастер и Маргарита",
      "author": "Михаил Булгаков",
      "chapter": 24,
      "chapter_title": "Извлечение мастера",
      "similarity": 0.97,
      "passage": "— Рукописи не горят. — …",
      "diff": [
        {"quoted": "рукописи не горят никогда", "source": "рукописи не горят"}
      ]
    }
  ],
  "message": null
}
```

- `matches` отсортированы по (−similarity, book_id, chapter); не больше `limit`
  (дефолт 3); совпадения с ratio < 0.75 в список не попадают.
- `not_found` → `matches: []`, `message` объясняет («ниже порога 0.75»; в режиме полки
  добавляет «FTS5 мог не найти сильно искажённую цитату — попробуй --book»).
- `passage` — подлинный текст книги (через карту смещений), расширенный до границ
  предложений, но не более ~400 символов.

### 4.2 CLI

```
lib verify "цитата" [--book ID] [--json] [--limit N]
```

Человеческий вывод: вердикт, локация (книга/глава), similarity, пассаж, diff построчно
(`- цитировано / + в книге`). Данные → stdout, диагностика → stderr (инвариант проекта).

Exit-коды: **0** = exact/close (цитата подтверждена), **1** = distorted/not_found
(не подтверждена — семантика grep), **2** = usage (пустая цитата; короткая цитата без
`--book`; несуществующий book_id).

### 4.3 MCP-тул `verify_quote` (шестой)

`serve.py`: тонкий адаптер `verify_quote(quote: str, book_id: str | None = None,
limit: int = 3) -> dict` — ноль своей логики, вызывает ядро. Описание тула — на
английском, как у остальных пяти, с явным «use this whenever you quote a book to the
user». `not_found` и «короткая цитата» — **валидные ответы** с `message`, не исключения
(паттерн `get --budget`): MCP-клиент всегда получает структуру.

## 5. Ошибки и края

- `book_id` — через существующий `validate_book_id` (traversal закрыт, MCP-безопасно).
- Цитата длиннее главы → окно = вся глава, сравнение целиком.
- Цитата через границу глав — не поддержана в v1; максимум — `distorted` по одной из
  половин. Документируем в README (Limitations).
- Пустая библиотека → `not_found` + message.
- `.search.db` нужен только режиму полки; режим книги работает и без FTS5
  (важно для системных Python без FTS5 — verify по книге доступен всегда).
- Markdown-заголовки глав вычищаются нормализацией → цитата, случайно включающая
  заголовок, всё равно матчится по телу.

## 6. Тесты

- **Юниты** (`tests/test_verify.py`): нормализация (кавычки/ё/тире/markdown),
  карта смещений, пороги вердиктов, tie-break, word-diff, «короткая цитата».
- **Интеграция на существующих фикстурах:** реальный абзац из golden-главы →
  точная цитата = `exact`; замена одного слова = `close` + diff ровно из одной пары;
  перестановка/пропуск клауз = `distorted`; чужой текст = `not_found`. Оба режима.
- **Детерминизм:** два одинаковых вызова → побайтово идентичный `--json` вывод.
- **MCP-смоук:** список тулов содержит 6, вызов `verify_quote` через stdio возвращает
  валидную структуру.
- **Golden:** не трогаются вообще (проверка — весь набор зелёный).

## 7. Вне скоупа v1

Мультиглавные цитаты; поиск перефразировок (семантика — принципиально не наша поляна:
нужен LLM); триграммный индекс для fuzzy-recall в режиме полки (доедем, если FTS5-фильтр
окажется узким местом на практике); проверка цитат по нескольким книгам сразу с
агрегацией.

## 8. Реализация — порядок

1. `verify.py`: нормализация + карта смещений (юниты).
2. Оконный матчер + вердикты + diff (юниты).
3. Режим полки через `search.py` (интеграция).
4. CLI-команда + exit-коды.
5. MCP-тул + смоук.
6. README (EN+RU): секция, таблица команд, Limitations; CHANGELOG; версия 0.2.0.
