# 📚 librarian

English version: [README.md](README.md) (канон, его видит PyPI).

Детерминированный конвейер «книга → Markdown-главы» для работы с LLM.
Любой входной формат (FB2, EPUB, DOCX, HTML, TXT/MD, PDF с текстовым слоем) →
каталог чистых глав с подсчитанными токенами. Без сети, без LLM, без случайности:
одинаковый вход даёт побайтово идентичный выход.

## Установка

    uv tool install librarian-cli     # или: pipx install librarian-cli

Офлайн-контракт: установка и рантайм не требуют сети (словарь токенизатора
вендорен в пакет).

## Быстрый старт

    lib ingest книга.epub роман.fb2 статья.html
    lib list
    lib list <book-id>
    lib get <book-id> 1-3,7
    lib get <book-id> --budget 12000
    lib find маятник
    lib find "closed timelike curve" --json

## Команды

| Команда | Назначение |
|---------|-----------|
| `lib ingest <файлы…> [--force] [--no-keep-source] [--config cfg.toml] [--verbose]` | Обработать книги в библиотеку; таблица «файл · id · статус · score». |
| `lib list [<book-id>]` | Без аргумента — все книги; с id — главы книги (n, заголовок, токены, summary). |
| `lib get <book-id> <spec>` | Печать глав по диапазону (`1-3,7`) в stdout. |
| `lib get <book-id> --budget N [--from K]` | Жадно подряд главы начиная с K, пока сумма токенов ≤ N. |
| `lib find <запрос> [--limit 10] [--book <id>] [--reindex] [--json]` | Полнотекстовый поиск по главам и названиям/авторам всей библиотеки (bm25, сниппеты, стемминг RU/EN). |
| `lib info <book-id>` | JSON: метаданные книги + метрики качества. |
| `lib doctor [<book-id>]` | Без id — книги в review и битые каталоги; с id — отчёт по книге. |
| `lib reingest --all [--config cfg.toml] [--verbose]` | Пересобрать библиотеку из `source/` текущим кодом/конфигом. |
| `lib rm <book-id>` | Удалить книгу и пересобрать индекс. |
| `lib serve [--library <путь>]` | MCP-сервер (stdio) поверх библиотеки. |
| `lib verify "цитата" [--book ID] [--json] [--limit N]` | Проверить цитату по книге или атрибутировать её по всей библиотеке. |

Корень библиотеки: `--library <путь>` (или переменная `LIB_HOME`, по умолчанию `./library`).
Данные идут в stdout, диагностика — в stderr. Коды выхода: `0` — успех, `1` — ошибка
выполнения, `2` — ошибка использования.

## Поиск и MCP

`lib find` строит производный индекс `.search.db` (SQLite FTS5) поверх готовых глав — он
вне контракта детерминизма (как `.lock`) и пересобирается из markdown; ingest о нём не
знает. Индекс лениво синхронизируется перед каждым запросом (правка книги → переиндексация
только её); `--reindex` — принудительная полная пересборка (спасательный люк, если правка
`.md` сохранила размер и mtime). Семантика AND: совпасть должны все слова; если точного
совпадения нет — автоматически частичное (OR) с пометкой. `--json` даёт ту же структуру,
что MCP-тул `find`.

**Ограничение поиска:** стемминг снимает регулярную флексию («поэзию» находит «поэзия»),
но не супплетивные формы («люди» ≠ «человек») и не опечатки.

### MCP-сервер

`lib serve` поднимает stdio-MCP-сервер (6 read-only тулов: `list_books`, `list_chapters`,
`find`, `get_chapters`, `book_info`, `verify_quote`) — Claude сам листает каталог, ищет и
вытаскивает главы под токен-бюджет, а свои цитаты может сверить с первоисточником:
детерминированный RAG без эмбеддингов, без сети, без API-ключей. `mcp` —
основная зависимость пакета (отклонение 37: реестр MCP не умеет extras, регистрирует
голый `uvx librarian-cli`), так что `serve` работает сразу после установки; extra
`librarian-cli[serve]` оставлен пустым как совместимый шим для старых команд.

Из установленного пакета (реестровый вход — отдельный консольный скрипт):

    claude mcp add librarian --env LIB_HOME=/путь/к/library -- uvx librarian-cli

В деве (из исходников репозитория):

    claude mcp add librarian -- uv run --directory /путь/к/repo lib serve --library /путь/к/library

Claude Desktop — тот же stdio-запуск в JSON-конфиге (см. `README.md`). Если пакет `mcp`
всё же отсутствует (битая установка), команда завершится подсказкой «пакет mcp отсутствует
— переустановите librarian-cli».

## Проверка цитат

*An MCP server that catches Claude misquoting your books.*

`lib verify "цитата"` проверяет, есть ли цитата в библиотеке дословно — пунктуация,
регистр, ё/е и markdown-разметка не в счёт. Два режима:

- **Режим книги** (`--book <id>`): полный скан одной книги — «это действительно есть в
  этой книге?». Работает без поискового индекса.
- **Режим полки** (без `--book`): атрибуция по всей библиотеке через FTS5-кандидатов —
  «откуда эта цитата?». Цитате нужно минимум 5 значимых слов (стоп-слова не считаются),
  иначе проверка пропускается с `verdict: null`.

```
lib verify "Рукописи не горят" --book bulgakov-master-i-margarita
lib verify "рукописи не горят никогда"
```

| verdict | смысл |
|---------|-------|
| `exact` | совпадение после нормализации (пунктуация/регистр/ё/типографика не в счёт) |
| `close` | similarity ≥ 0.95 — почти точная, расхождения показаны word-diff'ом |
| `distorted` | similarity ≥ 0.75 — это место, но цитата переврана |
| `not_found` | similarity < 0.75 — в книге/библиотеке такого нет |
| `null` | проверка не выполнялась: пустая цитата или короткая цитата без `--book` |

Коды выхода (семантика grep, осознанное отклонение от общепроектного «`1` — ошибка
выполнения» — см. `docs/MILESTONES.md`, отклонение 38):

| exit | случай | stdout |
|------|--------|--------|
| 0 | `exact` / `close` — цитата подтверждена | полный отчёт / JSON |
| 1 | `distorted` / `not_found` — не подтверждена | полный отчёт / JSON |
| 1 | рантайм-ошибка (неизвестный id книги, нет поискового индекса, занят индекс) | пустой |
| 2 | ошибка использования: пустая/короткая цитата (`verdict: null`), `--limit < 1` | пустой (message в stderr) |

Дискриминатор для скриптов: exit `1` с непустым stdout значит «проверено, не
подтверждена»; exit `1` с пустым stdout значит ошибку (диагностика ушла в stderr).
**Exit `0` включает `close`** — строгий CI-гейт должен проверять `verdict == "exact"` из
`--json`, а не сам код выхода.

## Качество

Каждая книга получает score из пяти метрик (coverage, structure, garbage, encoding,
dehyphen): `ok` (score ≥ 0.90 и без жёстких триггеров) — молча в библиотеку; `review`
(0.60 ≤ score < 0.90 либо есть триггеры) — сохранена с предупреждением, детали в
`lib doctor <id>`; `failed` (score < 0.60) — не сохраняется. Сканы и PDF под паролем
честно дают failed (OCR и снятие DRM — вне задач инструмента).

## Конфигурация

`lib ingest --config cfg.toml`; пороги и параметры резки — §14 спеки. Правки метаданных
руками: поставить `"meta_locked": true` в `book.json` — реингест их не перетрёт.
`lib reingest --all` пересобирает библиотеку текущим кодом/конфигом; книги, у которых
ничего не изменилось, пропускаются с сообщением «уже в библиотеке» — выход и так
детерминирован.

## Ограничения

- PDF: типографски обычные книги; сложная вёрстка может уйти в review.
- DRM не обходится; правомерность источников — ответственность пользователя.
- Сетевые ФС (NFS/SMB) не поддерживаются (advisory-lock ненадёжен).
- MOBI/DJVU/сканы-без-текста — нет (v3: OCR, calibre).
- `lib verify`: цитата через границу глав не поддержана (v1) — максимум `distorted` по
  одной из половин.
- `lib verify`: `close` при замене одного слова реалистичен от ~150 символов цитаты;
  короткая цитата с заменённым словом честно уезжает в `distorted` — треть трёхсловной
  цитаты и есть искажение.
- `lib verify` в режиме полки (без `--book`) требует поисковый индекс FTS5; режим книги
  (`--book <id>`) в CLI работает без него. `lib serve` целиком всё равно требует FTS5 —
  индекс синхронизируется на старте вне зависимости от того, какой тул вызовут.
- `lib verify` в режиме книги сканирует весь текст главы на чистом Python; длинная книга —
  секунды CPU. MCP-тул выполняет это в отдельном потоке, чтобы не блокировать остальной
  сервер.
